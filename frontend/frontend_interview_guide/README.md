## 简历定位

你的简历最强信号是：

- 有 Vue3 + Router + Pinia/Vuex 技术栈基础，项目里出现了路由权限、组件封装、状态同步和复杂交互。
- 有 AI 应用型前端项目，能讲 SSE 流式输出、`iframe` 实时预览、`postMessage` 跨域通信、DOM 监听注入和可视化编辑。
- 有协同编辑与可视化经验，能讲 WebSocket、编辑锁、冲突检测、ECharts 动态配置和数据看板。
- 有前后端联调与 AI 实习经历，能把前端和接口、异步任务、模型工作流、Docker/JMeter 等工程环节连起来。

当前最适合的投递方向：

- 第一优先：Vue 前端开发、B 端前端开发、AI 应用型前端、低代码/可视化搭建平台、数据可视化前端。
- 第二优先：前端工程化实习、全栈偏前端、Web 应用开发、协同编辑/在线工具方向。
- 谨慎投递：纯 C 端动效/强视觉岗位、纯 Node 后端岗位、纯算法岗。除非 JD 明确接受 AI 应用前端或全栈能力，否则需要额外补作品、工程化和基础深度。

## 总体补强策略

- 用项目深度弥补大厂实习或商业上线证据不足：每个项目都要能解释技术选择、状态流、异常处理、权限控制、性能瓶颈和替代方案。
- 用浏览器与网络基础托住前端关键词：SSE、WebSocket、`postMessage`、`iframe`、DOM 注入、响应式布局都要能讲到底层机制和安全边界。
- 用工程闭环区分“会写页面”和“能交付系统”：强调接口联调、错误处理、权限、日志、压测、部署、组件复用、测试和性能指标。
- 用算法题守住校招基本盘：前端校招仍常考手撕代码、JS 输出题和 LeetCode 中等题，不能只准备项目。
- 用 AI 前端作为差异化标签：不是泛泛写“会用 AI 工具”，而是讲清楚流式交互、实时预览、模型工具调用轨迹和人机协同编辑体验。

## 建议投递关键词

优先投递 JD 中出现这些词的岗位：

- Vue3、TypeScript、Vite、Pinia、Vue Router
- Ant Design Vue、Element Plus、ECharts、数据可视化
- WebSocket、SSE、实时协同、在线编辑、低代码、搭建平台
- AI 应用、AIGC、智能体平台、Prompt 调试台、模型调用、流式响应
- 前后端联调、B 端系统、管理后台、组件库、工程化

谨慎投递只强调这些词的岗位：

- Three.js / WebGL / 图形渲染
- React Native / 小程序 / Flutter
- Node.js 服务端深度开发
- 纯增长 C 端、强动效营销页
