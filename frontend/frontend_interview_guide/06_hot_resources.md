# 热门视频、博客与文档资源地图

## 使用原则

资源不需要全看。按你的目标岗位，优先级是：

1. 先看能直接服务面试的问题：JavaScript、CSS、浏览器、Vue3、手写题、算法题。
2. 再看能支撑项目深挖的官方文档：Vue Router、Pinia、Vite、MDN、web.dev、ECharts。
3. 最后看测试、工程化、低代码、AI 前端和面经，用来补差异化表达。

## Web 基础

| 资源 | 类型 | 适合阶段 | 怎么用 |
|---|---|---|---|
| [MDN Web Docs](https://developer.mozilla.org/zh-CN/) | 官方文档 | 全阶段 | 查 HTML、CSS、JavaScript、Web API、浏览器兼容性 |
| [MDN JavaScript](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript) | 官方文档 | JS 基础 | 查原型、Promise、模块、标准 API |
| [现代 JavaScript 教程](https://zh.javascript.info/) | 中文教程 | 基础补弱 | 按对象、函数、原型、Promise、模块系统复习 |
| [TypeScript Handbook](https://www.typescriptlang.org/docs/) | 官方文档 | TS 补强 | 重点看 narrowing、generics、utility types |

## Vue 生态

| 资源 | 类型 | 适合阶段 | 怎么用 |
|---|---|---|---|
| [Vue 官方文档](https://vuejs.org/) | 官方文档 | Vue3 主线 | 看响应式、Composition API、组件、生命周期 |
| [Vue Router](https://router.vuejs.org/) | 官方文档 | 路由权限 | 对照 `meta.access`、导航守卫、动态路由 |
| [Pinia](https://pinia.vuejs.org/) | 官方文档 | 状态管理 | 看 store、actions、插件和持久化思路 |
| [Vite](https://vite.dev/) | 官方文档 | 工程化 | 看 dev server、HMR、build、静态资源 |
| [VueUse](https://vueuse.org/) | 工具库 | hooks 思路 | 学 `useEventSource`、`useWebSocket` 等可组合逻辑 |

## 浏览器、网络与性能

| 资源 | 类型 | 适合阶段 | 怎么用 |
|---|---|---|---|
| [web.dev Core Web Vitals](https://web.dev/articles/vitals) | 官方性能资料 | 性能面试 | 理解 LCP、INP、CLS 和优化方向 |

| [MDN Server-sent events](https://developer.mozilla.org/zh-CN/docs/Web/API/Server-sent_events) | 官方文档 | SSE 项目 | 对照 `EventSource`、消息格式、连接限制 |
| [MDN WebSocket](https://developer.mozilla.org/zh-CN/docs/Web/API/WebSocket) | 官方文档 | 协同编辑 | 查连接、事件、发送和关闭 |

| [Can I use](https://caniuse.com/) | 兼容性查询 | 遇到 API 争议时查 | 查浏览器支持和降级方案 |

## 按岗位选择资源

Vue / B 端前端优先：

- Vue 官方文档、Vue Router、Pinia、Ant Design Vue、Element Plus、Vite。

AI 应用型前端优先：

- MDN SSE、MDN WebSocket、MDN postMessage、Dify Docs、OpenAI Streaming、LangChain JS。

- ECharts Handbook、ECharts Examples、web.dev 性能、Chrome DevTools。

- Vite、Vitest、Playwright、ESLint、Docker Docs、Chrome DevTools。
