这组文档围绕你的简历内容组织，不是通用八股清单。建议按下面顺序使用：

1. [学习主线与阶段规划](01_learning_mainline.md)：确定 12 周学习路线，覆盖测试理论、接口自动化、UI 自动化、性能测试、测试平台、AI 应用质量保障和游戏测试专项。
2. [快速补基础知识点](02_fast_foundation_notes.md)：把短期必须补齐的测试开发与游戏测试知识点整理成速查笔记。
3. [7 天短期冲刺复习](03_7day_sprint.md)：面试前一周使用，每天对应知识复习、练习任务、简历输出和自测问题。
4. [简历项目深挖模拟 QA](04_resume_deep_dive_qa.md)：围绕实习、接口自动化框架、持续测试平台、AI 应用测试和游戏测试迁移准备追问。
5. [常考知识点清单](05_common_knowledge_checklist.md)：作为查漏补缺表，按测试理论、接口、自动化、性能、CS 基础、游戏测试和 AI 质量保障分类。
6. [公开面经分析与面试打法](06_interview_experience_and_strategy.md)：整理公开资料中的面试共性，给出自我介绍、项目讲述、答题节奏和投递策略。

## 简历定位

你的简历最强信号是：

- 有测试开发主线：接口测试、Web UI 自动化、性能测试、稳定性测试、CI 报告和测试平台。
- 有工程化关键词：Python、Pytest、Requests、Postman、JMeter/Locust、Allure、Playwright/Selenium、Docker、MySQL、Redis、FastAPI、Vue.js。
- 有 AI 应用质量保障特色：Prompt/RAG 流程测试、模型服务接口联调、Golden Dataset、Bad Case 回流、多轮对话回归、JSON Schema/规则断言。
- 项目经历能支撑“测试工具开发”和“质量平台建设”，比单纯功能测试更适合测试开发岗位。

当前最适合的投递方向：

- 第一优先：测试开发工程师、自动化测试工程师、测试平台开发、AI 应用测试开发。
- 第二优先：游戏测试开发、游戏 QA 自动化、游戏服务端/客户端质量保障。
- 可尝试但需补强：纯游戏测试工程师、游戏性能测试、Unity/Unreal 自动化测试。简历里没有直接游戏项目，需要用“测试方法可迁移 + 游戏专项知识补强 + 自建练习案例”来弥补。

## 总体补强策略

- 把工具名升级成可讲的工程闭环：用例设计、数据构造、断言、执行调度、日志留存、报告分析、缺陷回归。
- 把 AI 测试讲成差异化优势：传统接口/UI/性能能力之外，突出不确定性输出、评估集、Bad Case、幻觉/拒答/格式校验。
- 面向游戏测试补一条专项线：玩法规则、状态机、网络弱网、兼容性、帧率/卡顿、崩溃日志、热更新、账号/背包/支付/匹配等高频模块。
- 每个项目都准备“为什么这样设计、怎么验证、出了问题怎么定位、如果重做怎么改”四层回答。

## 推荐输出物

建议在准备过程中沉淀这些材料：

- 一页测试开发知识脑图。
- 一个 Pytest + Requests 接口自动化最小框架。
- 一个 Playwright 页面对象模式小案例。
- 一份 JMeter/Locust 压测报告模板。
- 一份游戏测试用例集，例如登录、背包、抽卡、战斗结算、弱网重连。
- 一份项目架构图：接口自动化框架或持续测试平台。
- 一份 Bad Case 表：AI 回复错误、接口异常、UI 脚本不稳定、性能瓶颈、游戏断线/卡顿问题。
