## 简历定位

你的简历最强信号是：

- 有真实 RAG / Text-to-SQL 项目，涉及 Dify、LangChain、Milvus、MySQL、RAGAS、Golden Dataset 和 SQL 自纠错。
- 有客服场景的 LLM 应用实习经历，覆盖多轮记忆、槽位状态、Qwen 微调、Dense/BM25、RRF、Rerank。
- 有多模态识别与知识库入库系统，能讲 OCR/VLM、联网搜索、字段补全、去重、vLLM、FastAPI 落地。
- 有论文和竞赛，但面试中更建议把它们作为加分项，而不是主叙事。

当前最适合的投递方向：

- 第一优先：AI 应用开发工程师、LLM 应用工程师、RAG 工程师、Agent 应用工程师。
- 第二优先：大模型算法工程师、NLP 算法工程师、搜索/推荐中的 LLM 应用方向。
- 谨慎投递：纯模型训练岗、CV 算法岗、传统推荐算法岗。除非 JD 明确接受 LLM 应用经验，否则需要额外补实验、论文复现和模型训练深度。

## 总体补强策略

- 用项目深度弥补学校/实习品牌不足：每个项目都要能解释技术选择、指标口径、失败案例和替代方案。
- 用工程闭环区分“调包型 AI 应用”：强调数据、评估、接口、部署、压测、日志、反馈回流。
- 用底层原理托住简历关键词：RAG、LoRA、vLLM、KV Cache、Rerank、Text-to-SQL 都要能讲到原理和 trade-off。
- 用算法题守住校招基本盘：算法工程师面试通常仍会考手撕代码，不能只准备大模型应用。
