# 7 天短期冲刺复习

## 使用原则

这份计划适合面试前 7 天使用。每天分成四块：

- 知识复习：当天必须重新过一遍的概念。
- 手撕练习：当天算法题方向。
- 项目输出：把知识点和你的简历项目绑定。
- 自测问题：晚上用来模拟面试。

每天建议 6-8 小时。如果时间只有 3-4 小时，优先级是：项目深挖 > 算法题 > LLM/RAG 原理 > 传统八股。

## Day 1：数组、哈希、双指针、滑动窗口

### 知识复习

- 数组：连续内存、下标访问、原地修改。
- 哈希：用空间换时间，适合计数、去重、快速查找。
- 双指针：左右指针、快慢指针、同向指针。
- 滑动窗口：维护窗口内状态，常用于最长/最短子数组、字符串匹配。
- 前缀和：快速求区间和，常与哈希结合。

### 手撕练习

推荐题型：

- 两数之和。
- 三数之和。
- 最长无重复子串。
- 最小覆盖子串。
- 和为 K 的子数组。
- 合并区间。

解题要点：

- 先说暴力解，再说如何优化。
- 明确窗口什么时候扩大、什么时候收缩。
- 哈希表中 key/value 分别表示什么。
- 边界条件必须说清楚，例如空数组、重复元素、负数。

### 项目输出

把算法思维迁移到 RAG：

- 滑动窗口可以类比长文本 chunk 的窗口切分，但 RAG chunk 还要考虑语义完整性。
- 哈希可以用于文档去重、字段枚举值映射、缓存 key 设计。
- 前缀统计思想可以用于日志指标聚合，例如按时间统计成功率。

### 自测问题

- 滑动窗口适合什么题？不适合什么题？
- 哈希表为什么查询平均是 O(1)？最坏情况是什么？
- 你的知识库去重有没有用过 hash 或规则？如果没有，如何补充？
- RAG 中 chunk overlap 太大或太小分别有什么问题？

## Day 2：链表、栈队列、二叉树

### 知识复习

- 链表：节点引用、dummy head、快慢指针、反转。
- 栈：后进先出，适合括号匹配、递归模拟、单调栈。
- 队列：先进先出，适合 BFS。
- 二叉树：前中后序、层序、递归边界、路径问题。

### 手撕练习

推荐题型：

- 反转链表。
- 环形链表。
- 合并两个有序链表。
- 有效括号。
- 最小栈。
- 二叉树层序遍历。
- 二叉树最近公共祖先。
- 二叉树最大路径和。

解题要点：

- 链表题画指针变化，避免只背代码。
- 二叉树题先定义递归函数返回值。
- BFS 要说明队列中存什么。
- 单调栈要说明保持单调递增还是递减。

### 项目输出

把树和队列思维迁移到 Agent：

- Agent 工作流可以看成有状态图或任务树。
- 工具调用失败后的重试队列、异步任务队列、日志队列都可能被问到。
- LangGraph 的图式编排可以和状态机一起讲。

### 自测问题

- DFS 和 BFS 的区别是什么？什么时候选 BFS？
- 递归为什么可能栈溢出？如何改成迭代？
- 你的多 Agent 流程如何避免循环调用？
- 客服多轮对话中的状态流转能否画成状态机？

## Day 3：回溯、动态规划、图

### 知识复习

- 回溯：选择、递归、撤销选择、剪枝。
- 动态规划：状态定义、状态转移、初始化、遍历顺序。
- 图：邻接表、邻接矩阵、DFS/BFS、拓扑排序、并查集、最短路。

### 手撕练习

推荐题型：

- 全排列。
- 子集。
- 组合总和。
- 岛屿数量。
- 课程表。
- 最长递增子序列。
- 最长公共子序列。
- 编辑距离。

解题要点：

- DP 必须先讲状态定义，不能直接背转移方程。
- 图题要说明 visited 的作用。
- 拓扑排序要说明入度。
- 回溯剪枝要说明剪掉什么搜索空间。

### 项目输出

把 DP/图思维迁移到 Text-to-SQL：

- Text-to-SQL 链路可以拆成问题理解、Schema Linking、SQL 生成、执行校验、自修正。
- Schema 表之间的外键关系可以看成图，Join 路径选择可以看成图搜索。
- 多 Agent 流程中的依赖关系也可以用 DAG 表示。

### 自测问题

- 你的医药项目中 800 张表如何找到相关表？
- 如果问题涉及多张表 Join，你怎么确定 Join 路径？
- SQL 自修正最多重试几次？如何避免死循环？
- 项目工作流是 DAG 还是有环图？为什么？

## Day 4：机器学习与评估指标

### 知识复习

- 逻辑回归：线性模型、sigmoid、交叉熵。
- 决策树：信息增益、基尼系数、剪枝。
- 随机森林：Bagging、降低方差。
- GBDT/XGBoost：Boosting、拟合残差、特征重要性。
- 评估指标：Precision、Recall、F1、AUC、ROC、PR 曲线。
- 数据问题：样本不均衡、数据泄漏、过拟合、欠拟合。

### 手撕练习

推荐题型：

- 实现 sigmoid / softmax。
- 手写交叉熵。
- TopK 高频元素。
- 二分类阈值选择逻辑。
- 简单 KMeans 或 KNN 思路题。

### 项目输出

把 ML 评估迁移到你的项目：

- 医药项目：执行成功率、语法正确率、结果正确率、人工验收通过率要区分。
- 客服项目：业务采纳率不是模型离线指标，需要说明采纳定义、统计口径和样本周期。
- 检索项目：Hit Rate、Recall@K、MRR、NDCG、Rerank 前后对比要能解释。

### 自测问题

- Precision 和 Recall 如何取舍？
- AUC 的含义是什么？
- 你的 Hit Rate 提升 20% 是绝对提升还是相对提升？
- Golden Dataset 怎么构建才能避免数据泄漏？

## Day 5：Transformer、LLM 微调与推理

### 知识复习

- Self-Attention：Q/K/V、注意力分数、softmax、加权求和。
- Multi-Head Attention：不同子空间捕获不同关系。
- Position Encoding：绝对位置、相对位置、RoPE。
- Tokenizer：BPE、SentencePiece、中文分词问题。
- SFT：用高质量指令数据让模型学会任务格式和领域表达。
- LoRA：低秩矩阵训练权重增量，降低显存成本。
- PPO/DPO：对齐模型偏好，了解思想即可。
- KV Cache：缓存历史 K/V，提高自回归解码效率。
- vLLM：连续批处理、PagedAttention、服务化推理。

### 手撕练习

推荐题型：

- 手写 Attention 简化计算。
- Numpy 实现 softmax。
- Top-k / Top-p 采样解释。
- 给定 logits 计算交叉熵。

### 项目输出

准备客服项目答案：

- 为什么需要微调 Qwen，而不是只写 Prompt？
- 微调数据来自哪里？如何清洗？如何划分训练/验证？
- LoRA 参数如何选择？rank、alpha、target modules 分别影响什么？
- 幻觉如何评估和抑制？
- vLLM 部署前后延迟和吞吐如何对比？

### 自测问题

- Attention 为什么要除以 `sqrt(d_k)`？
- LoRA 适合注入知识还是适合调整行为风格？
- KV Cache 牺牲了什么换来了什么？
- 连续批处理如何提高 GPU 利用率？

## Day 6：RAG、Agent、Text-to-SQL

### 知识复习

- RAG 基本链路：数据清洗、切分、向量化、召回、重排、生成、评估。
- 混合检索：BM25 处理关键词匹配，Dense 处理语义相似。
- RRF：对不同召回列表按排名融合，降低不同分数尺度不可比的问题。
- Rerank：用更强模型对候选文档重新排序，提高精度但增加延迟。
- Query Rewrite：补全上下文、改写口语问题、扩展同义词。
- Agent Memory：短期上下文、长期摘要、结构化状态。
- Tool Calling：工具 schema、参数校验、异常处理。
- Text-to-SQL：Schema Linking、SQL 生成、执行校验、自纠错、安全控制。

### 手撕练习

推荐题型：

- 设计 LRU Cache。
- 设计 TopK 检索合并。
- 简单 BM25 公式理解。
- 给定两个排序列表，手算 RRF。

### 项目输出

准备医药 RAG 项目答案：

- 800 张表如何做 Schema 检索？
- 表名、字段名、字段含义、枚举值为什么要拼成富文本？
- RAGAS 指标怎么用？哪些指标适合问答，哪些不适合 Text-to-SQL？
- SQL 预执行如何捕获错误？如何把错误反馈给 LLM？
- 如何限制模型生成危险 SQL？

准备多模态项目答案：

- OCR 与视觉大模型如何分工？
- 联网搜索带来的幻觉如何控制？
- 字段置信度如何定义？
- 向量相似度和规则校验如何结合去重？

### 自测问题

- BM25 和向量检索各自的优势是什么？
- RRF 相比简单加权有什么好处？
- RAG 的召回率和生成正确性为什么不完全等价？
- Agent 为什么需要结构化状态，而不是只依赖上下文？

## Day 7：全真模拟与查漏补缺

### 上午：自我介绍与项目讲述

准备三个版本：

- 30 秒版：姓名、学校、方向、最强项目。
- 2 分钟版：教育背景、实习、两个项目、求职方向。
- 5 分钟版：重点展开一个项目，包含背景、技术方案、指标、反思。

推荐模板：

```text
我目前是重庆邮电大学计算机技术硕士，方向主要是 AI 应用开发和 LLM/RAG 工程。
我做过三个和岗位比较相关的项目：一个是医药数据分析场景的 RAG + Text-to-SQL 助手，一个是家电售后客服场景的小啄 GPT 智能回复模型，还有一个是多模态家电识别与知识库入库系统。
其中我最想展开的是医药问答项目，因为它涉及复杂 Schema 检索、SQL 生成、自纠错和评估闭环。这个项目面对 800 多张业务表，我主要负责 LLM 应用链路设计和 Schema 检索优化，最终把 SQL 执行成功率从约 70% 提升到 90%+。
```
