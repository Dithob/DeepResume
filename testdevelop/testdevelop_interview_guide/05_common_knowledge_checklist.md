# 常考知识点清单

## 使用方式

把这份清单当成面试前的打勾表。不要只看会不会背概念，还要看能不能结合你的简历项目讲出例子。

标记建议：

- `[x]` 能讲清原理、场景和项目例子。
- `[~]` 概念知道，但项目例子不稳。
- `[ ]` 还需要补。

- [ ] 软件测试的目标是什么，为什么测试不能证明软件没有缺陷。

## 手撕代码与题型

测试开发岗位通常会考简单到中等难度代码题，优先掌握：

- [ ] 数组和字符串：双指针、滑动窗口、前缀和。
- [ ] 哈希表：计数、去重、两数之和。
- [ ] 栈和队列：括号匹配、单调栈、队列模拟。
- [ ] 链表：反转、快慢指针、合并。
- [ ] 二叉树：遍历、层序、路径。
- [ ] 排序和二分。
- [ ] 简单动态规划。
- [ ] LRU Cache。
- [ ] TopK。

必须能回答：

- 时间复杂度和空间复杂度。
- 边界条件。
- 为什么选择这个数据结构。
- 如果输入规模很大怎么优化。
